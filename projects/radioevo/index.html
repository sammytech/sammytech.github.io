EVOLUTION RADIO

Firebase, Backendless, Java, AndroidSDK, Swift, PHP, HTML5, CSS3

Goals

  • To allow users to listen to the radio station
  • To enable users to contribute through a comment section

Requirements

  1. Register users with email
  2. Register users with their Facebook account
  3. Login users with their email and password
  4. Login users with their Facebook account
  5. Modify information about a student borrower in the system (NOTE: Only “Administrator” workers can mark a student borrower as “delinquent”, or remove the “delinquency” mark)
  6. Play internet radio station
  7. Live updates of a comment section
  8. Users reply to other users.
  9. Change of in-app artwork to reflect radio station current program.
  10. Notify users when certain radio station programs start.
  11. Notify users when another user replies to their comment.

Development

  • Due to this app being made as a prototype for the start-up radio station, free subscription to some of the technologies were used. Third-party solutions were also used to reduce cost of making the mobile application.

  • Technologies

    • Firebase – This platform was used for creating the live comment section in the app because it is a real-time database and it allows for syncing data between multiple device easy.
    • Backendless – This MBaaS was used because Parse was being shut down. It was used for push notifications to users.
    • Android SDK / Java – This is the language and software development kit necessary for creating android application
    • Swift – This is the language used for creating the iOS version of the application.
    • PHP – This was the language used to create the backend of the web application which allowed administrators to manually push notifications and change artworks in the app.
    • HTML5 – This was the language used to create the backend of the web application which allowed administrators to manually push notifications and change artworks in the app.
    • CSS3 – This was used to style the user interface of the web application
  • iOS - All user interfaces were coded instead of using the interface builder because I felt I had more control on what happens and what is being displayed in the app.
  • Android - All user interfaces were coded using the XML.
